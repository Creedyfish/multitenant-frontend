# LogistiCore Frontend

Multi-tenant inventory and supply-chain frontend built with TanStack Start, React 19, and TanStack Query/Router.

## What is implemented

### 1. Multi-tenant authentication and session lifecycle

- Login flow with workspace/subdomain + email/password.
- Access token kept in memory (not localStorage) for better security posture.
- Refresh token handled through HttpOnly cookie.
- Silent refresh on app boot to restore sessions.
- Automatic 401 handling in the API client:
  - runs a single refresh request
  - queues concurrent failed requests
  - retries original requests after refresh
  - forces logout if refresh fails
- Logout mutation clears token, auth store, and query cache.

### 2. Role-aware app shell and permissions

- Protected authenticated layout for all internal routes.
- Role-based navigation and feature access (ADMIN, MANAGER, STAFF).
- Centralized permission flags via auth hooks to avoid scattered role checks.
- Collapsible sidebar + topbar user menu/sign-out flow.

### 3. Feature modules and API integrations

The app is organized by feature under src/features with TanStack Query hooks for each domain.

- Dashboard
  - KPI cards: total products, pending PRs, warehouse count, low-stock alerts.
- Products
  - List/search/paginate and create/update/delete operations.
- Purchase Requests
  - Full lifecycle support: create, update, submit, approve, reject, mark ordered, receive.
- Stock Movements
  - Ledger with filters, stock levels, and mutations for in/out/transfer/adjust.
- Warehouses
  - List/create/update/delete and per-warehouse stock-level views.
- Suppliers
  - List/create/update/delete.
- Users
  - List/create/update/delete.
- Audit Logs
  - List/detail with filters (entity/action/actor/date range).
- Settings
  - Profile update.
  - Password change.
  - Organization settings (admin-only tab).

### 4. Observability and production readiness

- Sentry client integration with:
  - Router browser tracing.
  - Session replay.
  - Error capture.
- Server-side Sentry tunnel route to reduce client-side DSN exposure and bypass ad blockers.
- Global request/function middleware integration for TanStack Start.

## Local development

### 1. Install dependencies

npm install

### 2. Environment variables

Create .env.local in the project root.

Required/used variables:

- VITE_API_BASE_URL
  - Backend base URL for API calls.
  - Default fallback in code: http://localhost:8000
- VITE_SENTRY_DSN
  - Sentry DSN (required if Sentry is enabled).
- VITE_APP_TITLE
  - Optional app title value.
- SERVER_URL
  - Optional server-side env from env.ts.

Example:

VITE_API_BASE_URL=http://localhost:8000
VITE_SENTRY_DSN=https://<key>@o0.ingest.sentry.io/<project>
VITE_APP_TITLE=LogistiCore

### 3. Start dev server

npm run dev

App runs at http://localhost:3000.

## Scripts

- npm run dev: Start development server (with dotenv + instrumentation).
- npm run build: Build production output.
- npm run preview: Preview production build.
- npm run start: Run built server output.
- npm run test: Run test suite with Vitest.
- npm run lint: Run ESLint.
- npm run format: Check formatting with Prettier.
- npm run check: Apply Prettier + ESLint fixes.
- npm run storybook: Start Storybook on port 6006.
- npm run build-storybook: Build Storybook static output.
